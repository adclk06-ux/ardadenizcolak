# Project Structure

**Source of Truth:** Product DNA, Information Architecture, ADRs  
**Purpose:** Define where every file lives, why it lives there, and what does not belong.  
**Scope:** Source directory only. Build artifacts, dependencies, and configuration files are documented elsewhere.  

---

## Root: `src/`

The `src/` directory contains everything that is authored, edited, or curated by hand. Nothing in `src/` is generated by a build step, fetched from an API, or downloaded by a package manager. If a file is removed from `src/`, it is gone from the platform.

---

## `src/components/`

Reusable UI units. Each component is a single concern. Components are not pages. Components do not fetch data. They receive data and render it.

**Subdirectories:**

### `components/layout/`
Page-level structural components that appear on every page or every page of a specific type.

| Component | Purpose | Used By |
|---|---|---|
| `BaseLayout.astro` | Root HTML wrapper (`<html>`, `<head>`, `<body>`), global styles injection, metadata | All pages |
| `PageHeader.astro` | Persistent navigation, active state logic, skip-to-content link | All pages |
| `PageFooter.astro` | Minimal footer: home link, system docs link, copyright | All pages |
| `Main.astro` | Semantic `<main>` wrapper with consistent padding and max-width | All pages |

**What does not belong:** Page-specific sections, content blocks, or anything that appears on only one destination.

### `components/navigation/`
Navigation-specific components. Isolated because navigation is a primary interaction surface and merits its own directory.

| Component | Purpose |
|---|---|
| `PrimaryNav.astro` | The six persistent navigation items. Active state via current URL path comparison. |
| `MobileNav.astro` | Collapsed toggle for viewports below threshold. Same six items, no simplification. |
| `NavItem.astro` | Single navigation link with active/inactive state styling. |

**What does not belong:** Breadcrumbs (not used per Information Architecture). Dropdown menus (not used). Scroll-spy logic (not used).

### `components/typography/`
Text components that enforce the type scale and prevent arbitrary sizing.

| Component | Purpose | Maps to Token |
|---|---|---|
| `Display.astro` | Page titles, section headers | `--type-display` |
| `Body.astro` | Primary reading text | `--type-body` |
| `Caption.astro` | Metadata, labels, secondary info | `--type-caption` |
| `Mono.astro` | Code, technical specs, data | `--type-mono` |
| `Prose.astro` | Article/case study body text wrapper with default line height and measure |

**What does not belong:** Inline style overrides, `font-size` props, or any component that accepts arbitrary sizing.

### `components/project/`
Components specific to the Projects destination (`/work`).

| Component | Purpose |
|---|---|
| `ProjectList.astro` | Index page: renders the list of case study summaries |
| `ProjectCard.astro` | Single summary item: title, problem, outcome, lifecycle state |
| `ProjectDetail.astro` | Full case study layout: Problem → Context → Decision → Outcome → Reflection |
| `LifecycleBadge.astro` | Reusable state label (Published, Archived, Deprecated, etc.) |

**What does not belong:** Thumbnail galleries, client logo bars, skill percentage indicators.

### `components/note/`
Components specific to the Technical Writing destination (`/notes`).

| Component | Purpose |
|---|---|
| `ArticleList.astro` | Index page: renders the list of article summaries |
| `ArticleCard.astro` | Single summary item: title, thesis, category |
| `ArticleDetail.astro` | Full article layout with editorial structure |
| `ArticleMeta.astro` | Date (subtle), category, lifecycle state |

**What does not belong:** Tag clouds, "related posts" algorithms, social share buttons, comment sections.

### `components/primitives/`
Layout primitives. Structural components that compose space. Not destination-specific, not decorative.

| Component | Purpose |
|---|---|
| `Container.astro` | Max-width wrapper with responsive padding. |
| `Section.astro` | Full-width section with configurable vertical padding. |
| `Stack.astro` | Vertical flex with controlled gap and alignment. |
| `Cluster.astro` | Horizontal flex-wrap with gap and justification. |
| `Grid.astro` | CSS grid with column count and gap. Collapses to 1 column on mobile. |
| `Center.astro` | Horizontal centering with optional text-align and max-width. |

**What does not belong:** Modals, tooltips, dropdowns, accordions, carousels, tabs, pagination (not used in this platform).

### `components/motion/`
Animation wrappers that apply the Animation Policy (ADR-0004). Every component here must pass the motion audit: *does this motion communicate information?*

| Component | Purpose | Trigger |
|---|---|---|
| `FadeIn.astro` | Opacity + subtle translateY on page load | Page mount |
| `HoverState.astro` | CSS transition wrapper for interactive elements | Hover |
| `FocusRing.astro` | Visible focus state for keyboard navigation | Focus |

**What does not belong:** Scroll-triggered animations, parallax wrappers, custom cursor components, loading spinners (use inline text per ADR-0004).

## `src/layouts/`

Page templates that compose components into complete pages. Layouts are not components — they are compositions.

| Layout | Purpose | Used By |
|---|---|---|
| `Base.astro` | HTML document shell, metadata, global styles, header, footer | All pages |
| `Homepage.astro` | Base + minimal density, condensed identity, navigation only | `/` |
| `ProjectIndex.astro` | Base + project list layout, medium density | `/work` |
| `ProjectDetail.astro` | Base + case study layout, high density | `/work/[slug]` |
| `ArchitectureIndex.astro` | Base + architecture entry list, medium density | `/systems` |
| `ArchitectureDetail.astro` | Base + architecture entry layout, high density | `/systems/[slug]` |
| `NoteIndex.astro` | Base + article list, low-medium density | `/notes` |
| `NoteDetail.astro` | Base + article layout, medium density | `/notes/[slug]` |
| `About.astro` | Base + factual layout, medium density | `/about` |
| `Contact.astro` | Base + minimal layout, no scroll | `/contact` |

**What does not belong:** Logic for fetching data, content parsing, or state management. Layouts receive already-parsed content.

## `src/pages/`

File-system routing entries. Each file in `pages/` corresponds to a URL path. These files are thin — they parse content, select a layout, and render.

| File | URL | Responsibility |
|---|---|---|
| `index.astro` | `/` | Render homepage: identity statement + navigation |
| `work/index.astro` | `/work` | Query all work content, sort by relevance, render ProjectIndex layout |
| `work/[slug].astro` | `/work/project-name` | Query single work entry by slug, render ProjectDetail layout |
| `systems/index.astro` | `/systems` | Query all systems content, render ArchitectureIndex layout |
| `systems/[slug].astro` | `/systems/entry-name` | Query single systems entry by slug, render ArchitectureDetail layout |
| `notes/index.astro` | `/notes` | Query all notes content, render NoteIndex layout |
| `notes/[slug].astro` | `/notes/article-name` | Query single note entry by slug, render NoteDetail layout |
| `about.astro` | `/about` | Render About layout with static content |
| `contact.astro` | `/contact` | Render Contact layout with static content |

**What does not belong:** Business logic, data transformation, or content authoring. Pages are routing surfaces, not application logic.

## `src/content/`

All Markdown content, organized by destination. This is the authoritative content layer. Content is not duplicated elsewhere.

```
src/content/
├── work/
│   ├── project-name.md
│   └── another-project.md
├── systems/
│   ├── ci-cd-pipeline.md
│   └── monitoring-strategy.md
├── notes/
│   ├── article-name.md
│   └── another-essay.md
├── about.md
└── contact.md
```

**Content File Rules:**
- Every `.md` file has YAML front matter per ADR-0006.
- File names become URL slugs. Use hyphens, not underscores or camelCase.
- Content in `project` state (front matter) is excluded from build queries.
- No nested directories within content folders. Flat organization.

**What does not belong:**
- Drafts, `_drafts` folders, or unpublished content in the repository.
- Images (those belong in `assets/`).
- JSON or YAML data files (those belong in `lib/` if structured, or in front matter).

## `src/styles/`

The styling system per ADR-0003. Plain CSS with custom properties. No preprocessor. No utility framework.

| File | Purpose |
|---|---|
| `tokens.css` | Design tokens: colors, type scale, spacing, breakpoints. The single source of truth. |
| `reset.css` | Minimal CSS reset: normalize browser defaults without opinionated overrides. |
| `base.css` | Global element styles: body, headings, paragraphs, links, lists, code. |
| `layout.css` | Layout primitives: grid system, max-width, padding, margin utilities (limited). |
| `utilities.css` | Minimal utility classes for unavoidable one-offs (e.g., `.visually-hidden`). |
| `motion.css` | Animation definitions per ADR-0004: keyframes, transitions, reduced-motion block. |

**What does not belong:**
- Component-specific styles (those live with components as `<style>` blocks or scoped CSS).
- Media queries outside of `tokens.css` (breakpoint values are tokenized).
- Arbitrary values or one-off colors.

---

## `src/assets/`

Static files that are referenced but not authored as text.

| Subdirectory | Contents |
|---|---|
| `assets/images/` | WebP screenshots. Flat, no device frames. Max 1200px width. |
| `assets/diagrams/` | SVG diagrams. Inline where possible, external file where reused. |
| `assets/fonts/` | Self-hosted font files (WOFF2) if not using system or CDN fonts. |

**What does not belong:**
- Stock photography.
- Icon fonts.
- Video files.
- AI-generated imagery.
- Files larger than 200KB without explicit justification.

---

## `src/lib/`

Utility functions that do not render UI. Pure logic.

| File | Purpose |
|---|---|
| `content.js` | Content querying: parse Markdown, validate front matter, filter by state, sort by relevance. |
| `urls.js` | URL helpers: slug generation, path construction, internal link validation. |
| `format.js` | Formatting utilities: date formatting (when shown), number formatting, text truncation. |
| `metadata.js` | Site-wide metadata: title templates, descriptions, OG image paths. |

**What does not belong:**
- Framework-specific code (React hooks, Vue composables). The platform is framework-agnostic in its logic layer.
- API clients. There are no APIs.
- State management. There is no application state beyond the current page.

## Directory Boundaries

These rules prevent the slow erosion of structure into a flat or disorganized mess:

**1. Components do not import layouts.**  
The dependency flows one way: layouts compose components. Components never reference layouts.

**2. Pages do not import components directly.**  
Pages select a layout. The layout composes components. Pages are routing surfaces, not UI assemblies.

**3. Content does not import components or styles.**  
Markdown files are pure content. They may reference assets (images, diagrams) but they do not include UI logic.

**4. Styles do not reference content or assets.**  
CSS is generic. It describes how things look, not what they contain.

**5. `lib/` does not import from `components/`, `layouts/`, or `pages/`.**  
Logic is independent of presentation. If a utility needs UI context, it is misclassified.

**6. No `src/utils/` or `src/helpers/` catch-all directories.**  
Everything in `lib/` has a specific, named purpose. Generic utility directories accumulate orphaned functions.

## What Is Not in `src/`

These items live at the project root, not in `src/`:

| Item | Location | Reason |
|---|---|---|
| Build configuration | Root (`astro.config.mjs` or equivalent) | Build tool convention |
| Package dependencies | `package.json`, `node_modules/` | Dependency management |
| CI/CD config | `.github/workflows/` | Platform automation |
| Public static files | `public/` | Files copied as-is to build output (robots.txt, favicon.ico) |
| Documentation | `docs/` | Meta-content about the platform itself |
| Product DNA | `PRODUCT_DNA.md` | Strategic foundation |
| Information Architecture | `INFORMATION_ARCHITECTURE.md` | Structural contract |
| ADRs | `docs/adr/` | Decision record |
